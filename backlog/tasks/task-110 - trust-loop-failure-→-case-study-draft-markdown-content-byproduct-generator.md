## Контекст

Хотелка: каждая сессия работы рождает один артефакт (пост / разбор / demo). Сейчас контент-байпродукт требует ручной работы: переписать context, скопировать spec snippet, отформатировать curl, объяснить classification. Это трение убивает поток.

Ровно для этого нужен генератор: failure-id → markdown черновик. Не финальный пост, а 80%-готовый draft, который остаётся отредактировать и опубликовать.

## Что сделать

Команда: `zond report case-study <failure-id> [-o draft.md]`. Альтернатива — кнопка «Generate case-study draft» в UI Run detail на failure-card.

Markdown шаблон:

```
# <method> <path> — <short description of failure>

## TL;DR

- Что нашли: <classification> (definitely_bug / likely_bug / quirk)
- Что пошло не так: <гайдлайн на одну строку>
- Repro: см. ниже

## Context

API: <spec title + version>
Endpoint: <method> <path>
Provenance: тест сгенерирован <generator>, ветка спеки <ref>

### What the spec says

<JSON pointer + fenced YAML/JSON snippet из OpenAPI>

## Repro

\`\`\`bash
<curl command>
\`\`\`

## What happened

Status: <status>
Response:

\`\`\`json
<response body>
\`\`\`

## Why it matters

<если definitely_bug — описание из failure-evidence>
<если quirk — описание ожидания и фактического поведения>

## How zond found it

<тип probe + параметры запуска>

---
*Generated by zond <version> from run <run-id>*
```

Поля заполняются из существующих данных (provenance, classification, evidence, spec snippet). Если каких-то полей не хватает — оставлять placeholders с явными `<TODO: ...>` чтобы пользователь видел что доредактировать.

## Acceptance

- `zond report case-study <failure-id>` создаёт markdown файл по шаблону.
- Шаблон заполнен реальными данными из SQLite + OpenAPI.
- TODO-placeholders вставлены там, где данных нет.
- Кнопка «Generate case-study draft» в UI копирует тот же markdown в clipboard.
- Работает для failure_class = definitely_bug / likely_bug в первую очередь.
- Документация в ZOND.md.

## Стратегическая ценность

Превращает contend-by-product из «ещё одна работа» в «отредактировать существующий черновик». Прямой enabler хотелки «один артефакт за сессию». Усиливает дефенсибельность каждой находки — у тебя готовый текст для тикета/поста/слака.
