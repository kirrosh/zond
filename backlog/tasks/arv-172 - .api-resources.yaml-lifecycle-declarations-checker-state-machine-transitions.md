## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Цель D из m-20.

Расширение .api-resources.yaml: per-resource lifecycle блок.

Пример:
resources:
  subscription:
    lifecycle:
      field: status
      states: [pending, active, cancelled, past_due]
      transitions:
        - from: pending
          to: [active, cancelled]
        - from: active
          to: [cancelled, past_due]
        - from: past_due
          to: [active, cancelled]
        - from: cancelled
          to: []  # terminal
      actions:
        cancel:
          endpoint: POST /v1/subscriptions/{id}/cancel
          expected_state: cancelled

Новый checker (zond checks --check lifecycle):
1. После каждого action endpoint'а из manifest'а — GET resource, проверить state-field.
2. Двойной cancel → 4xx или idempotent 2xx с тем же state (HIGH если 5xx или state регрессирует).
3. Обратные переходы (cancelled → active) → HIGH.
4. Skip-transitions без objявленного пути → MEDIUM.

Acceptance:
- apis/stripe/.api-resources.yaml содержит lifecycle для ≥1 ресурса (subscription или payment_intent).
- Checker зелёный на корректной траектории; ловит induced violation в тесте (forced double-cancel via raw request).
- Manifest schema валидируется на дефекты (циклы, недостижимые states, missing terminal).
