## Контекст

Источник: [m-9 feedback round 5](../notes/m-9-workspace-hygiene/feedback-original.md), §P9-P10.

После ~6 раундов экспериментов в воркспейсе скапливаются stale
файлы: 4 × `security-digest-v*.md`, 4 пустых `security-emit-v*/`,
264 probe-yaml после bug-fix'ов в шаблонах. Нет команды
«regenerate from scratch» или «убрать всё, что zond сгенерил».
Плюс пользователь не может отличить «свой» файл от auto-generated.

## Что сделать

1. **Manifest.** `.zond/manifest.json` (или `.yaml`) на уровне
   воркспейса со списком сгенерённых путей и их sha256:

   ```json
   {
     "version": 1,
     "generated": [
       { "path": "apis/sentry/.api-catalog.yaml", "sha256": "...", "by": "zond add api", "ts": "..." },
       { "path": "apis/sentry/probes/validation/probe-...yaml", "sha256": "...", "by": "zond probe-validation --emit", "ts": "..." }
     ]
   }
   ```

   Каждая `zond add api` / `generate` / `probe-* --emit` дописывает
   запись. Если файл изменён руками (sha256 не совпадает) —
   `zond clean` его не трогает.

2. **`zond clean` команда:**
   - `zond clean --api <name>` — удалить всё auto-generated для API
     (probes/, generated tests, не трогая `.env.yaml` и ручные правки).
   - `zond clean --probes` — только probe-suites.
   - `zond clean --all` — весь auto-generated контент в воркспейсе.
   - `--dry-run` (по дефолту) — показать что удалится, не удалять.
   - `--force` — реально удалить.

3. **Header в auto-файлах.** Сейчас `.api-catalog.yaml` имеет
   `# Auto-generated by zond. Regenerate with: …`. Распространить на
   все probe-suites, generated tests:

   ```yaml
   # Auto-generated by zond probe-validation v0.x.x
   # Run: zond probe-validation --api sentry --emit ...
   # ⚠️ Edits will be overwritten on regenerate. Remove from .zond/manifest.json to keep.
   ```
